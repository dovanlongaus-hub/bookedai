# BookedAI SaaS Domain Foundation

This file is a concise companion to `target-platform-architecture.md`.

It exists to preserve a simple truth for future phases:

- SEO should be attributable to revenue
- matching should be explicit and explainable
- booking should be confidence-aware
- payment should be trust-aware
- CRM should own relationship lifecycle
- BookedAI should own operational truth
- mobile should be designed for early

When future work changes one of these areas, that work should be checked against the architecture baseline before implementation.
