# Future Swim Content And Copy Approval Checklist

Date: `2026-04-20`

Document status: `active tenant content approval checklist`

## Purpose

This checklist is the final content and copy approval pass for the Future Swim tenant runtime before production launch.

Use it alongside:

- `docs/development/future-swim-tenant-use-case.md`
- `docs/development/future-swim-launch-runbook.md`

## 1. Brand and tone approval

Confirm the site reads as:

- professional
- modern
- child-safe
- parent-reassuring
- swim-school specific

Reject if the site reads like:

- a generic SaaS landing page
- a marketplace directory
- a chatbot-first experience with weak swim-school positioning

## 2. Homepage copy approval

Approve or revise:

- hero headline
- hero supporting paragraph
- primary CTA label
- secondary CTA label
- trust framing near the top of page
- language about children aged `2-6`

Confirm:

- the message is clear for parents
- the message emphasizes swim learning and safety
- the message does not overclaim live automation that is not yet configured

## 3. Program and learning-path copy

Check that:

- age bands are understandable
- progression language is calm and confidence-building
- water-safety language is appropriate
- parent expectations are realistic

Reject if:

- copy sounds clinical or robotic
- copy implies guaranteed outcomes
- copy implies a class structure not actually supported by Future Swim

## 4. Location content approval

For each seeded Future Swim location, verify:

- centre name is correct
- suburb spelling is correct
- address is correct
- description is acceptable
- no outdated venue detail is shown

Required locations:

- Caringbah
- Kirrawee
- Leichhardt
- Miranda
- Rouse Hill
- St Peters

## 5. Assistant language approval

Confirm the receptionist assistant:

- sounds like a helpful booking and enrolment assistant
- does not sound too technical
- does not suggest external providers
- does not overstate schedule certainty
- uses safe enquiry and callback language when appropriate

Check:

- welcome message
- shortlist explanation
- no-result or weak-result message
- enquiry confirmation wording

## 6. Booking and enquiry wording approval

Approve:

- enquiry form labels
- selected-centre language
- booking-intent language
- email follow-up wording
- support or callback wording

Confirm:

- `request`, `enquiry`, or `callback` wording is used where live slot confirmation is not guaranteed
- wording does not imply real-time class confirmation unless integration truth supports it

## 7. Contact and support content

Confirm:

- support email is correct
- support labels are correct
- any follow-up expectation language is approved
- any parent-facing reassurance copy is approved

## 8. Visual-content approval

Confirm:

- imagery is suitable for a preschool swim-school audience
- visuals do not look stock-heavy or off-brand
- section art supports trust and safety
- visual emphasis stays on swim-school quality, not on generic AI branding

## 9. Compliance and truthfulness check

Confirm:

- no false claim about availability
- no false claim about automation readiness
- no false claim about CRM sync already being live if it is pending
- no false claim about email automation being active if SMTP is not configured

## 10. Final sign-off record

Record:

- reviewer name
- review date
- approved with no changes
- approved with small edits
- hold for revision

Recommended sign-off owners:

- product owner
- tenant-facing operator
- Future Swim business approver if available
